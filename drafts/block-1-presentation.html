3 Princípios
Que Validamos

1

Context Engineering
é a nova core skill

  • Contexto estruturado vs ad-hoc = diferença enorme de qualidade
  • Warmup (carregar contexto uma vez) vs repetir tudo a cada prompt
  • Repository + reference examples = output consistente

Evidência:

32.7% → 80%+

Diferença de aceitação está no contexto

2

Responsabilidade aumenta,
não diminui

De

"Escrever código"

→

Para

"Validar código +
arquitetar contexto"

IA não reduz carga cognitiva, redistribui
Ownership se torna mais crítico

Evidência (LinearB):

PRs de IA são 2.6x maiores

PRs de IA levam 5.3x mais tempo de review

3

Conhecimento técnico
permanece fundamental

  • Entender arquitetura é necessário para prompting efetivo
  • Inglês vira "linguagem de programação" mas profundidade técnica é mandatória
  • Não é substituição, é amplificação de quem já sabe

Evidência:

Times com melhor performance técnica
têm melhores resultados com IA
