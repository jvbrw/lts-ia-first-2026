# Bloco 1: O Que Aprendemos Construindo Produto com IA em 2025
**Apresentador: JV**
**Duração: ~30 minutos**

---

## Objetivo

Compartilhar aprendizados reais, não visão ideal. Responder à pergunta central:

> "O que 2025 nos ensinou, na prática, sobre construir produto com IA no centro?"

---

## Estrutura

### 1. Abertura (2min)
- Pergunta interativa sobre expectativa vs realidade com IA
- Dados de contexto: 88.3% adoção, 32.7% aceitação
- Setup do problema: a ferramenta não é o mais importante, é como a empresa se organiza para usar a/as ferramentas disponíveis [setup organizacional]

### 2. Onde IA funcionou bem (5min)
- **Padrão**: Funciona quando há contexto estruturado e uma lógica de captura de valor em cada etapa. IA como fluxo de trabalho ao invés de trabalho atomizado
- **Caso 1**: POCs de viabilidade técnica
- **Caso 2**: Features de baixa-média complexidade
- **💬 Interação**: "Onde funcionou pra vocês?"

### 3. Onde IA quebrou expectativas (5min)
- **Anti-padrão**: Delegar sem estrutura
- **Caso 1**: "Cria feature pra mim" - feature que ninguém usou e nem validou se faz sentido
- **Caso 2**: AI PR rejeitado - 3x maior, 5x mais review, overwork
- **Caso 3**: Excesso de agentes sem clareza de valor
- **💬 Interação**: "Onde não bateu expectativa?"

### 4. AI Framework - Nossa resposta prática (10min)
- O que é e por que criamos
- 5 capabilities: PM, Tech Lead, SWE, QA, Refinement
- Fluxo: initiative-start → refinement → implement → qa-test
- TDAID: Test-Driven AI Development
- Resultado: de 32% para +80% de aceitação
- **💬 Interação**: "Dúvidas sobre o framework?"

### 5. 3 Princípios validados em 2025 (5min)
- **Context Engineering** é a nova core skill
- **Human-in-the-Loop** muda de lugar, não desaparece
- **Repertório** importa mais que prompt
- **💬 Interação**: "Qual ressoa mais?"

### 6. Dados de mercado - Contexto 2026 (3min)
- LinearB 2026 Benchmarks
- Adoção alta ≠ resultado alto
- Contexto é o gargalo
- 45% não medem impacto
- **💬 Interação**: "Vocês medem?"

### 7. Transição para Bloco 2
- O problema não é ferramenta, é organização
- Setup para Bassan falar sobre redesenho de times

---

## Pontos-Chave a Enfatizar

1. **Dados externos validam nossa experiência** - LinearB mostra que o problema é sistêmico
2. **AI Framework como prova** - Não estamos só falando, construímos e usamos
3. **Context engineering é o diferencial** - Quem estrutura melhor, tem melhor resultado
4. **Medição é rara mas crítica** - 45% operando no escuro
