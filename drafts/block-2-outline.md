# Bloco 2: O Que Esses Aprendizados Exigem do Desenho dos Times em 2026
**Apresentador: Gustavo Bassan**
**Duração: ~30 minutos**

---

## Objetivo

Construir diretamente a partir dos aprendizados do Bloco 1. Responder à pergunta central:

> "O que líderes precisam redesenhar agora para operar produto IA-first em 2026?"

---

## Estrutura

### 1. O Shift Fundamental: SOR → SOI (5min)
- Systems of Record vs Systems of Intelligence
- Tabela comparativa: dashboards → inteligência, human-in → human-on
- O modelo mental que precisa mudar
- **💬 Interação**: "Onde vocês estão nessa transição?"

### 2. Redesenhando Ownership (7min)
- Por que ownership tradicional não funciona com IA
- O problema: Quem é "dono" de output gerado por IA?
- **3 novas responsabilidades:**
  - Context Owner - Garante input
  - Validation Owner - Valida output
  - Feedback Owner - Captura aprendizados
- **💬 Interação**: "Como ownership funciona hoje?"

### 3. Context Engineering como Disciplina (8min)
- Por que virou disciplina, não tarefa
- **Product Spec como Infraestrutura** - Pré-requisito, não nice-to-have
- **Warmup como Prática Padrão** - Carregar contexto antes de pedir output
- **Documentação como Input** - Shift de mindset
- Exemplo do AI Framework: warmup-tech.md, warmup-project.md
- **💬 Interação**: "Documentação é input ou output pra vocês?"

### 4. Evitando IA por Conveniência (5min)
- Anti-padrão: "Vamos criar um agente pra isso"
- Framework de decisão: contexto, critério, volume, expertise
- 4 perguntas antes de implementar IA
- **💬 Interação**: "Vocês têm framework de decisão?"

### 5. O Papel do Líder em Times IA-First (5min)
- De Executor para Orquestrador
- De Especialista para Arquiteto de Contexto
- De Gerente de Pessoas para Gerente de Sistema
- Métricas que importam: input, throughput, learning
- **💬 Interação**: "Como o papel de vocês mudou?"

### 6. O Que Não Pode Mais Ser Adiado (3min)
- **Decisão 1**: Definir ownership de contexto
- **Decisão 2**: Estabelecer métricas de IA
- **Decisão 3**: Investir em context engineering
- **💬 Interação**: "Qual é mais urgente pra vocês?"

### 7. Encerramento conjunto com JV
- Takeaways dos dois blocos
- O que fazer segunda-feira
- Q&A

## Pontos-Chave a Enfatizar

1. **Transição SOR → SOI é inevitável** - Questão é quando, não se
2. **Ownership precisa ser explícito** - Não pode ser implícito com IA
3. **Context engineering é investimento** - Não custo
4. **Líder vira arquiteto de sistema** - Não executor principal

## Framework Conceitual

### Systems of Intelligence (Activant Capital)

| De (SOR) | Para (SOI) |
|----------|------------|
| Dashboards estáticos | Inteligência em tempo real |
| UIs rígidas | Plataformas headless multi-modal |
| Human-in-the-loop | Human-on-the-loop |
| Human middleware | Autonomous outcomes |
| Software que informa | Software que age |

### Os 3 Owners em Times IA-First

| Owner | Responsabilidade | Métrica |
|-------|------------------|---------|
| Context | Garantir input | Qualidade input → output |
| Validation | Validar output | Taxa de aceitação, tempo review |
| Feedback | Capturar aprendizados | Evolução ao longo do tempo |
